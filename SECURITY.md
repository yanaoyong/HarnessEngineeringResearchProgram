# Security Policy

## 私有报告安全问题

请不要为凭据泄露、workflow injection、校验器绕过、恶意文件、隐私数据或其他可被利用的问题创建公开 Issue。

使用 GitHub 的 [私有安全通告入口](https://github.com/yanaoyong/HarnessEngineeringResearchProgram/security/advisories/new) 提交报告。若该入口在当前仓库不可用，请通过 repository owner 的 GitHub profile 联系维护者，并且不要在公开消息中粘贴 secret、token 或个人数据。

报告应尽量包含：

- 受影响的文件、commit 或 workflow；
- 可复现条件和最小复现步骤；
- 潜在影响与攻击边界；
- 已知缓解方式；
- 是否包含需要立即轮换的凭据或私有数据。

## 协调披露

维护者会按影响范围尽力确认、复现和协调修复，但当前项目不承诺固定响应 SLA。修复完成前，请避免公开可利用细节。

## 不属于安全报告的事项

拼写、断链、研究内容争议、Source Registry 更新和计划实验建议应使用公开 Issue Form。研究结论错误只有在造成可利用的安全影响或敏感信息暴露时才进入私有安全渠道。
