## 文档定位

V4.2 不是固定课程表，而是一套 Cycle-based 研究导航。每个 Cycle 围绕一个核心问题建立 Mental Model，使用有明确权限边界的 Evidence 提出 Hypothesis，通过最小实验观察结果，再更新设计判断。

```text
Research Question
        ↓
Mental Model V0
        ↓
Authorized Evidence
        ↓
Falsifiable Hypothesis
        ↓
Small Experiment
        ↓
Observation
        ↓
Mental Model V1
        ↓
Design Judgment / Next Question
```

## 使用规则

1. Cycle 是研究单元，不是日历门禁。
2. 每个 Cycle 最多一个核心 Research Question。
3. 每 2–4 个 Cycle 做一次 Route Review，并记录路线为何改变。
4. 开源项目按研究问题和 Evidence authority 使用，不按仓库顺序漫游。
5. Stable Task Suite 用于方向性比较，不用于公开 benchmark。
6. 新 Experiment 使用 `EXP-Cxx-yy`，并为每次执行保存 run metadata。
7. Host、Provider、Model 与 Configuration 必须分别记录。
8. 结论必须标明 Evidence ID、适用范围、限制和验证日期。
9. Support Level 按 capability scope 评定，不给 Host 做无边界总评分。
10. `Unsupported` 与 `Unknown` 必须区分。
