## 文档定位

**这不是一份“16 周后必须全部打勾”的课程清单，而是一份研究导航。** 每个 Week 是一个研究循环：围绕一个核心问题建立初步认知，用优秀开源项目和官方资料理解机制，再通过小规模实践攻击自己的认知，最后根据证据修正 Mental Model，并形成对 myharness 的设计判断。

| **核心原则：** 研究问题优先于课程顺序；证据优先于印象；小实验优先于大重构；发现新问题可以调整路线，但当前循环最多只追一个核心问题。 |
|-------------------------------------------------------------------------------------------------------------------------------------|

## 贯穿全程的研究—实践螺旋

```text
Research Question 研究问题
↓
Build Mental Model 建立初步认知
↓
Hypothesis 提出假设
↓
Small Experiment 小规模实践
↓
Observe Evidence 观察证据
↓
Update Mental Model 修正认知
↓
Design Judgment 形成设计判断
↓
Next Question 进入下一个问题
```

## 使用规则

1.  “周”是研究单元，不是时间门禁。4 天想清楚可以提前走；真实迁移问题暴露关键矛盾时，可以借用后续阶段的方法。

2.  每周最多一个核心研究问题。旁支问题记录到 Open Questions，除非它直接否定当前实验，否则不立即展开。

3.  开源项目按 L3 深拆、L2 定向研究、L1 对照样本三级深度阅读。没有研究问题，不打开对照项目。

4.  具体目录路径是当前导航锚点，不是永恒 Contract。进入 Claude Code、Codex 或大型开源项目阶段前，快速刷新官方 Changelog、Feature Maturity 或默认分支。

5.  前 12 周可以做小实验和局部验证，但尽量避免因“看到优秀设计”直接大改 myharness；架构集成集中在 Week 13–16。

## 项目阅读深度

| **研究对象**                                   | **阅读深度** | **本周只关注**                                 |
|------------------------------------------------|--------------|------------------------------------------------|
| minimal-agent-tutorial / mini-swe-agent        | L3 深拆      | Agent Core、Loop、Environment、History、Action |
| learn-claude-code                              | L3 深拆      | Harness Primitive、教学重实现、机制假设        |
| Codex 定向源码                                 | L3 深拆      | 与 Harness 接口相关的 crate 与稳定 Surface     |
| Superpowers                                    | L3 深拆      | Skill 行为、验证、开发方法                     |
| Claude 官方 / ACE / Spec Kit / BMAD / Agent OS | L2 定向      | 只研究当前核心问题对应能力                     |
| OpenSpec / OpenHarness / Ralph / SWE-agent ACI | L1 对照      | 用于回答单一问题，不做仓库漫游                 |

## 16 周总体路线

| **阶段** | **周次** | **主题**                       | **阶段目的**                                                         |
|----------|----------|--------------------------------|----------------------------------------------------------------------|
| Phase 1  | 1–2      | Agent / Harness Foundation     | 建立 Agent Core 与 Harness Primitive 基础心智模型                    |
| Phase 2  | 3–4      | Claude Code Host Model         | 理解 Context 生命周期与 Extension / Control Surface                  |
| Phase 3  | 5–6      | Codex Host Model               | 定向源码阅读，建立执行、安全、状态与定制模型                         |
| Phase 4  | 7        | Cross-host Harness Abstraction | 区分 Portable Core、Host Adapter、Host-specific Capability           |
| Phase 5  | 8–12     | Harness Engineering Research   | 研究 Skill、Change、Workflow、Context、Knowledge 与 Minimalism       |
| Phase 6  | 13–16    | myharness Integration Research | Audit → Hypothesis / ADR → Minimal Experiment → Acceptance / Beliefs |

## 整个计划真正期待的成长

| **阶段** | **你开始习惯追问的问题**                         |
|----------|--------------------------------------------------|
| Week 1   | Agent 是怎么工作的？                             |
| Week 4   | 为什么这个约束应该放 Hook，而不是 Rule / Skill？ |
| Week 7   | 这个能力是 Host-specific 还是 Portable？         |
| Week 10  | 这个 Gate 控制了什么风险？                       |
| Week 12  | 这个问题值得增加 Harness 复杂度吗？              |
| Week 14  | 我的架构判断能否被证伪？                         |
| Week 16  | 我的证据是什么？                                 |

**最终目标不是“读过 Claude Code、Codex、Superpowers 和 Spec Kit”，而是形成一套稳定的研究能力：**

- 观察 Agent 行为。

- 建立机制模型。

- 从优秀项目提炼 Pattern。

- 提出可证伪的设计 Hypothesis。

- 设计最小实验并保留证据。

- 根据 Evidence 修正认知。

- 解释为什么 myharness 应该做、暂缓做或不应该做某件事。

## 路线调整说明

本卷是研究导航，不是冻结的教学脚本。执行到对应研究循环前，应先刷新相关官方文档、默认分支与 Changelog；若项目目录或能力名称发生变化，继续追踪本卷定义的研究问题与 Capability，而不是机械寻找旧路径。
