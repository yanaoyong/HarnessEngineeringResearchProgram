## 1. 决策（Decision）

自 V4.3 起，四个主要 Coding Agent Host 是：

1. Claude Code
2. Codex
3. Qwen Code
4. OpenCode

Qwen Code 取代 ZCode，成为 Cycle 7、Cycle 9 与 Cycle 18 四宿主研究矩阵中的主要对象。该替换是 repository owner 的路线优先级决策，不是由已执行 benchmark、Support Assessment 或企业验证推出的产品排名。

## 2. 保持不变的协议（Unchanged Protocol）

- 18 个 Cycle 的编号、顺序与 Batch 1–8 边界；

## 4. 四宿主研究角色（Four-host Research Roles）

| Host | V4.3 Research Role |
|---|---|
| Claude Code | mature Harness best-practice reference |
| Codex | advanced open-source implementation reference |

Qwen Code 与 OpenCode 都可以研究多 Provider / Model 条件，但研究角色不同：Qwen Code 重点观察国内模型生态、Qwen-first integration、官方开源实现与组织部署事实；OpenCode 重点观察 vendor-neutral Host semantic 与 Provider / Model portability。角色差异不是能力高低结论。

## 5. Qwen Code 来源权限（Source Authority）

Qwen Code 可以使用：

- Official Contract；
- verified Official Source；
- official release information；
- Direct Behavior Evidence；
- Project Evidence；
- local configuration；
- Enterprise Fact。

`QwenLM/qwen-code` 已由官方文档与官方组织身份确认是 Qwen Code 的官方源码仓库。该身份事实不等于任意默认分支内容已经成为可复现 Source Evidence。形成源码判断前仍必须：

1. 固定完整 commit；
2. 记录 repository、path、question、search term、stop point 与 limitation；
3. 将实际执行的 package / binary / release artifact 映射到对应 tag 或 commit；
4. 分开登记 Contract、Source 与 Behavior；
5. 不把源码实现自动外推到其他版本、surface、Provider 或 Model。

默认分支、浮动官方文档与最新 release 只作为执行时重新核验的 anchor。当前内容迁移不创建 `EVD-*`，也不声称任一安装 artifact 已映射到固定 commit。

## 6. Cycle 7 当前边界（Current Cycle 7 Boundary）

Cycle 7 研究：

- Qwen Code CLI 与明确绑定的其他 surface；
- `QWEN.md` / memory、configuration、tools、approval / permission、sandbox、hooks、skills、subagents、extensions 与 session / artifact route；
- official source 中与上述 capability 直接相关的 bounded implementation path；
- Qwen / Alibaba Cloud、OpenAI-compatible、Anthropic-compatible、Gemini-compatible 或其他已授权 Provider profile 的分层；
- installation、identity、credential、network、telemetry、logging、upgrade / rollback、support 与 procurement 的 deployment-specific Enterprise Fact。

Cycle 7 不执行公开模型 benchmark，不把 Qwen Code 与 Qwen Model 混为一体，不把多协议支持写成行为可移植性，不创建通用企业或法律结论，也不实现 myharness Qwen Code Adapter。

## 7. 迁移完整性（Migration Integrity）

- Cycle 7 继续使用 `EXP-C07-01` 与 `EXP-C07-02`，因为实验尚未执行，不存在需要保留的 Run / Result / `EVD-*`。
- Cycle 9 与 Cycle 18 的第三个 Host stratum 由 ZCode 改为 Qwen Code；原有 Experiment ID 排列不变。
- 已删除的 `SRC-ZCODE-*` 没有派生 `EVD-*`，因此不会造成 Evidence Claim 断链。
- 任何关于“Qwen Code 是综合首选”的判断在获得本计划自己的绑定证据前，只作为 owner route decision，不进入 Evidence Claim 或 Support Level。
